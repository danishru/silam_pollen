Зачем это нужно?

Данные о пыльце там, где уже живут твои автоматизации

От прогностической модели к понятному состоянию Home Assistant

SILAM Pollen помогает добавить контекст пыльцы в панели, уведомления и сценарии Home Assistant без отдельного ключа доступа или платного аккаунта.

🔑
Без ключей доступаИспользует открытые прогнозные данные SILAM от Финского метеорологического института.
🏠
Настройка по зонеМожно использовать зону Home Assistant или вручную скорректировать координаты и высоту.
🛡️
Контроль данныхПроверка покрытия, статус службы, восстановление из кэша и полное обновление помогают понимать поведение интеграции.

Под реальные ежедневные решения

  • Оставить окна закрытыми при высоком уровне берёзы или злаковых?
  • Будет ли завтра лучше для прогулки или проветривания?
  • Сломался внешний сервис или выбранная точка вне покрытия?
  • Какой аллерген отвечает за повышенный индекс?

Прогнозы носят информационный характер и не заменяют медицинские рекомендации, но хорошо подходят как контекст для умного дома и личных сценариев.

Сущности

Что создаёт интеграция

🌾

Сенсоры концентрации аллергенов

Отдельные сенсоры для выбранных типов пыльцы, обновляемые через один общий координатор и доступные как обычные сущности Home Assistant.

ОльхаБерёзаЗлаковыеОрешникПолыньОливаАмброзия
📈

Индекс и ответственный аллерген

Устаревший индекс может показывать сводное состояние, основной аллерген и прогноз на завтра, если включён режим прогноза.

🧪

Диагностика для отладки

Статус службы, время запроса, горизонт прогноза, данные кэша, тип запроса и выбранный набор данных помогают понять последнее обновление.

Режим прогноза

Текущие значения, прогнозные окна и погодные карточки

Одна прогнозная сущность, несколько представлений

При включённом режиме прогноза интеграция создаёт погодную сущность для условий пыльцы. Она может отдавать почасовой прогноз, прогноз дважды в сутки и ежедневный прогноз на основе объединённого ответа SILAM.

С учётом стоимости обновлений

Прогнозные запросы тяжелее текущих значений, поэтому интеграция использует общий координатор, восстановление из кэша, постепенную догрузку и отдельную службу полного обновления, когда свежие данные нужны явно.
