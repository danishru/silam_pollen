Why use it?

Pollen context where your automations already live

From forecast model to clear Home Assistant state

SILAM Pollen brings pollen context into Home Assistant dashboards, notifications, and routines without a separate access key or paid account.

🔑
No access keysUses openly available SILAM forecast data from the Finnish Meteorological Institute.
🏠
Zone-based setupUse a Home Assistant zone or manually adjust coordinates and altitude.
🛡️
Data controlCoverage checks, service status, saved-data recovery, and full refresh help explain integration behavior.

Built around real daily choices

  • Should the windows stay closed during a high birch or grass period?
  • Is tomorrow likely to be better for a walk or ventilation?
  • Is the external service unavailable, or is the selected point outside coverage?
  • Which allergen is driving the elevated index?

Forecasts are informational and do not replace medical guidance, but they are useful context for smart home routines and personal planning.

Forecast mode

Current values and short forecast views

One forecast entity, several views

When forecast mode is enabled, SILAM Pollen creates a Home Assistant weather entity for pollen conditions. It can show near-term changes, day and night blocks, and daily context from the same SILAM data.

Designed to update responsibly

Forecast requests can be heavier than current values, so the integration uses shared updates, saved-data recovery, incremental refresh paths, and a full refresh service when fresh data is explicitly needed.
